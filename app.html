Javascript Promisehandling

This app will explain the basic JS promise behavior

  • What causes the first promise callback function to execute (resolve callback)?
  • What causes the second promise callback function to execute (reject callback)?
  • What causes the catch callback function to execute?
  • Shows that Promises can handle a finally, which is a relatively new feature (console output)

The Source code holds 2 functions

  1. The "low level" function (promiseFunktion) returns a promise that immedially resolves or rejects, based on a numerical Value
  2. The "high level" function (showResult) resolves that promise. Based on a boolean Value it throws an error, inside the promise callbacks

Please analyze the source code. Together with the following parameter explanation. The basic Promise handling will be very clear with this.

Each of the following buttons represents an individual parameterization.

  • "1" for whatToDoInPromise (Promise Resolve)
  • "False" for callCatch (no Exception)
  • "2" for whatToDoInPromise (Promise Reject)
  • "False" for callCatch (no Exception)
  • "1" for whatToDoInPromise (Promise Resolve)
  • "True" for callCatch (throw Exception)
  • "2" for whatToDoInPromise (Promise Reject)
  • "True" for callCatch (throw Exception)
